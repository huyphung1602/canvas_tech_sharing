Creating Games Without a Game Engine

A deep dive into game development fundamentals

1. The Problem

"I would like to create a game without a game engine."

But why? Understanding the fundamentals helps us:

  • Learn how game engines work internally
  • Create lightweight games without overhead
  • Have complete control over the implementation

3. Basic Game Loop

while (true) {
    // Game logic here
}

❌ Problem: This will freeze the browser!

4. Fixed Time Step

The Problem

How to make sure the re-rendering JS only runs after a fixed time instead of CPU time?

❌ Bad Solution

setInterval(gameLoop, 16); // ~60 FPS

✓ Better Solution

requestAnimationFrame(gameLoop);

The Challenge

const deltaTime = currentTime - lastTime;

6. Game Loop Structure

while (true) {
    processInput();
    update();
    render();
}

processInput()

Handle user input (keyboard, mouse, etc.)

update()

Update game state, physics, etc.

render()

Draw the current game state

8. Frame Timing

while (true) {
    double start = getCurrentTime();
    processInput();
    update();
    render();

    sleep(start + MS_PER_FRAME - getCurrentTime());
}

In JavaScript with Canvas:

const MS_PER_FRAME = 1000 / 60; // 60 FPS

function gameLoop(timestamp) {
    const deltaTime = timestamp - lastTime;
    if (deltaTime >= MS_PER_FRAME) {
        update(deltaTime);
        render();
        lastTime = timestamp;
    }
    requestAnimationFrame(gameLoop);
}
